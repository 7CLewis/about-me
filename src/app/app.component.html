PiRate

GitHub Page

This is a standalone, customizable security testing tool implemented on a Raspberry Pi 3 device.

Technologies Used

  • Angular 11
  • NodeJS
  • HTML5
  • CSS3
  • ExpressJS
  • Python 3
  • PrimeNG

My Role

I was tasked with building and designing the entire frontend of the project, and connecting it to function properly with our NodeJS backend and Python controller.

Project Difficulties and Solutions

I have gained quite a bit of experience in Angular over the past few years, so the real challenges came when trying to connect it properly to the backend, and getting the backend to call the Python controller to run the tests specified by the user on the frontend. I ended up deciding to make the Node Express Server spawn a subprocess that ran the Python controller with the user-specified test list, and then having the server listen to stdout where the controller would return the results of the executed tests.


Notable Features

  • Ability to create and add your own custom security test (i.e., port scan, specific vulnerability scan, etc.).
  • Data storage in browser's Local Storage, with option to save data to file.
  • Works on all major Operating Systems.

Meevee

GitHub Page

Chrome extension to automatically open Zoom meetings.

Technologies Used

  • Angular 10
  • Angular Material UI
  • HTML5
  • SCSS/CSS3
  • Chrome Extension Management
  • Chrome Storage API

My Role

For this Chrome extension, I designed the "Add Event" page, and implemented the logic for the validate, add, delete, and edit event functions.

Project Difficulties and Solutions

No one on our team had ever created a browser extension before, so we all had to spend extensive time reading through documentation and learning how it worked. We also ran into issues throughout the project with getting Chrome to open when no other browser windows were currently open, and getting the alarm to hit at the right time. We solved the first issue by making sure our extension had the proper permissions, and the second issue by storing all times in UTC, so that there wouldn't be any timezone issues.

SAAST

GitHub Page

Shodan API Automated Search Tool for Linux

Technologies Used

  • Python 3
  • Shodan
  • Cron
  • SMTP over SSL
  • Bash

My Role

I designed this project alone. The project allows users to enter a Shodan query and create a crontab for it, so the query runs repeatedly and alerts the user via email when any of their specified notification parameters are hit.

Project Difficulties and Solutions

Shodan's free version unfortunately has quite a few limitations to it, so it was difficult to find tests that didn't require a paid membership. Once I did find some tests, however, I was able to get it to run successfully. Another issue is that there are so many different tests a user could run and so many different parameters they would want to specify to be notified about, so I had to attempt to make the notification options as broad as possible, while still ensuring that it would work properly.


Notable Features

  • Automated email notification system.
  • Crontab creation.
  • Automated bash script creation and execution.
  • Interaction with the Shodan API.

Smash Shell

GitHub Page

A simplified Bash shell written in C

Technologies Used

  • C
  • Make
  • Bash

My Role

Solo project. I created a simplified Bash shell to learn how it works.

Project Difficulties and Solutions

Developing in C means figuring out segmentation faults and memory leakage. There was a lot of manual memory allocation required for this project, so I had to make sure to free that memory at the correct times to avoid early frees and memory leakage. Process forking is also required to redirect file descriptors, so I had to make sure to wait for that process to finish to avoid having a zombie process.


Notable Features

  • Command piping
  • I/O redirection
  • Manual memory allocation
  • File opening & closing, and reading & writing
  • Command history saving and viewing
